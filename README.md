# 智能跌倒检测与看护系统 (AI Vision Guardian)

> 基于 **YOLOv8 + MediaPipe Pose + PyQt6 + MySQL** 的桌面端人员跌倒实时检测系统，支持**管理员运维端**与**家属亲情看护端**双角色登录，提供跌倒告警、视频存档、飞书实时推送、健康指标可视化大屏等功能。

> 本项目适用于智慧养老、智能安防、毕业设计等场景；检测结果不应作为医疗诊断或唯一的安全决策依据。

---

## 界面预览
系统采用 **One Dark 现代暗黑科技风**，融合了侧边栏菜单、指标卡片、实时视频流、健康大屏等元素，整体观感专业且符合运维/看护人员的长时间使用需求。

---

## 功能特性

### 核心检测
- YOLOv8 实时定位画面中的人员（COCO `person` 类）
- MediaPipe Pose 提取人体肩部、髋部等关键点
- 根据躯干倾角三段式识别：**站立** / **跌倒中** / **平躺**
- 自适应阈值：`< 35°` 站立，`35° ~ 70°` 跌倒中，`> 70°` 平躺

### 告警与存档
- 检测到跌倒立即播放 `alarm.mp3` 警报
- 自动拼接**前置缓冲区**视频片段，落盘至 `result/` 目录
- 告警事件写入 **MySQL 数据库**，留存历史可回溯
- 通过 **飞书机器人 Webhook** 推送可视化告警卡片

### 双角色体系
- **管理员运维端**：开放摄像头切换、系统高级设置、阈值调参等所有能力
- **家属亲情看护端**：精简菜单，新增 **飞书音视频通话** 与 **健康指标大屏**

### 历史与处理
- 内置历史跌倒记录回放器，支持在程序内选择片段播放
- 家属处理意见可填写备注并标记"已处理"，同步写回数据库

### 其他特性
- 中文字体自适应绘制（`msyh.ttc` / `simhei.ttf`）
- 异步线程处理告警：音视频写入、数据库、飞书推送互不阻塞
- One Dark 主题：扁平化按钮 + 圆角卡片 + 高亮交互反馈

## 技术原理

1. 使用 YOLOv8 检测每帧中的人员。  
2. 对人员区域 (bbox) 进行 MediaPipe Pose 姿态估计。  
3. 用左右肩、左右髋的中点计算**躯干倾角**。  
4. 根据三段式阈值判断姿态。  
5. 当"跌倒中"状态连续出现若干帧 → 触发告警：保存视频 → 写库 → 推送飞书；人员站起或平躺稳定若干秒后结束录制。

## 项目结构（多文件拆分版）

```text
Mediapipe-Fall-Detector/
├── main.py              # 应用入口 + 主窗口 + 角色权限分发
├── ui_layout.py         # One Dark 风格主界面布局 (Ui_MainWindow)

├── detector.py          # YOLOv8+MediaPipe 检测线程 (DetectionThread)
├── database.py          # MySQL 数据库初始化 (users / alarm_logs)
├── yolov8n.pt           # YOLOv8 Nano 权重 (需自行准备)
├── alarm.mp3            # 警报音频 (可替换为自定义音频)

├── result/              # 自动生成的告警片段目录
├── requirements.txt     # Python 依赖清单
└── README.md            # 本说明文档
```

## 快速开始

### 1. 克隆与虚拟环境

```bash
git clone https://github.com/<your-username>/Mediapipe-Fall-Detector.git
cd Mediapipe-Fall-Detector

python -m venv .venv
.venv\Scripts\activate

pip install -r requirements.txt
```

### 2. 准备模型与音频

将 YOLOv8 Nano 权重文件放在项目根目录，并命名为 `yolov8n.pt`。首次使用可从 [Ultralytics YOLOv8](https://github.com/ultralytics/ultralytics) 获取；若网络可用，Ultralytics 通常会在权重缺失时尝试自动下载。

将告警音频命名为 `alarm.mp3` 放在项目根目录（可自行替换）。

### 3. 确保 MySQL 已启动

默认配置（`database.py`）：

- host: `localhost`
- user: `root`
- password: `231006410`
- database: 启动时自动创建 `fall_detector_db`

首次运行会自动创建数据库与表，并写入默认账号。

### 4. 启动应用

```bash
python main.py
```

## 使用说明

### 管理员运维端

1. 以 `root` 登录。
2. 在左侧选择**摄像头**或**导入视频文件**作为视频源。
3. 点击**启动检测**，右侧可观察实时姿态、告警卡片与日志。
4. **系统高级设置**：调节躯干倾角阈值、声音告警开关、录制前置缓冲区。
5. **历史片段**：在程序内查看历史告警、下载/打开存档目录。

### 家属亲情看护端

1. 以 `family` 登录。
2. 实时监测能力与管理员一致，但部分参数被锁定。
3. **飞书音视频通话**：一键调起飞书客户端，向老人发起音视频对话。
4. **健康指标大屏**：模拟心率/血氧/血压/步数/体温/电量等指标（演示用，后期对接硬件）。
5. 在**历史片段**中可填写处理备注并标记"已处理"。

## 注意事项

- 程序默认摄像头索引 `0`；如有多摄像头可在主窗口左上角下拉切换。
- 中文绘制依赖 `msyh.ttc` 或 `simhei.ttf`，若显示异常请安装相应字体或修改 `detector.py` 中字体路径。
- 姿态阈值、连续帧判定时长影响灵敏度与误报率，建议针对实际安装高度、拍摄角度进行现场调参。
- 飞书 Webhook URL 当前**强绑定**于源码，可在 `main.py` / `detector.py` / `dialogs.py` 中查找替换。
- `alarm.mp3`、`result/`、大体积测试视频、`yolov8n.pt` 建议不提交到 Git；可使用 Releases 或外部网盘分发。
- `database.py` 含明文数据库密码，**仅供本地演示**；正式部署请改用环境变量或加密配置文件。

## 后续可扩展方向

- **硬件联动**：通过 MQTT/TCP 接入智能手环、毫米波雷达数据，补齐真实健康指标

- **云端**：将 MySQL 替换为云数据库，支持跨地域家属随时查看

- **语音对讲**：集成 WebRTC，实现检测到跌倒后自动外呼家属
